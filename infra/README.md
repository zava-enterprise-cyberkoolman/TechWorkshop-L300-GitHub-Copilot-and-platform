# ZavaStorefront – Azure Infrastructure

Generated by GitHub Copilot to address the infrastructure requirements for deploying the **ZavaStorefront** ASP.NET Core MVC application to Azure.

## Architecture Overview

| Resource | SKU / Tier | Purpose |
|---|---|---|
| **Resource Group** | — | Single container for all dev resources |
| **Azure Container Registry** | Basic | Stores Docker images; no admin credentials |
| **Log Analytics Workspace** | PerGB2018 | Centralised log sink |
| **Application Insights** | Workspace-based | App monitoring & telemetry |
| **App Service Plan** | Linux B1 | Compute for the web app |
| **App Service (Web App)** | Linux container | Hosts ZavaStorefront Docker container |
| **AI Foundry Hub** | — | AI platform hub |
| **AI Foundry Project** | — | Project scoped to the hub |
| **Azure OpenAI / AI Services** | S0 | GPT-4 + Phi-3 model deployments |

**Region:** `westus3` (required for GPT-4 and Phi model availability)

## Security

- App Service uses a **System-Assigned Managed Identity** to pull images from ACR via the `AcrPull` RBAC role — no passwords or admin credentials stored.
- ACR admin user is **disabled**.
- App Service enforces **HTTPS only**.

## Deploying

```bash
# 1. Login
az login
azd auth login

# 2. Initialise environment
azd init

# 3. Preview the deployment
azd provision --preview

# 4. Provision infrastructure + deploy app
azd up
```
